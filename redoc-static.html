Приложение для трекинга финансов (1.0.0)

Download OpenAPI specification:

Backend для android приложения для трекинга финансов.

API позволяет:

  • Создавать и управлять транзакциями (доходы и расходы)
  • Получать статистику по расходам за различные периоды
  • Управлять категориями транзакций
  • Настраивать повторяющиеся транзакции

Все эндпоинты требуют аутентификации через JWT токен.

Statistics

Получение статистики по расходам

Получить статистику по расходам за период

Возвращает статистику по расходам за указанный период. Если параметры from и to не указаны, возвращает статистику за текущий месяц. Если указан только один параметр, возвращает ошибку 400.

Authorizations:
bearerAuth
query Parameters
from
string <date>
Example: from=2025-09-01

Дата начала периода в формате YYYY-MM-DD. Если не указана вместе с to, используется текущий месяц.

to
string <date>
Example: to=2025-09-30

Дата конца периода в формате YYYY-MM-DD. Если не указана вместе с from, используется текущий месяц.

Responses

Production server

http://spendings-pages.ddns.net/api/statistics

Development server

http://localhost:8080/api/statistics

Transactions

Управление транзакциями

Получить список транзакций

Возвращает список транзакций с возможностью фильтрации по категориям и датам

Authorizations:
bearerAuth
query Parameters
category
Array of strings
Example: category=food&category=transport

Фильтр по категории (можно указать несколько)

from
string <date>
Example: from=2025-09-01

Дата начала периода в формате YYYY-MM-DD. Если не указана, фильтрация по дате не применяется.

to
string <date>
Example: to=2025-09-30

Дата конца периода в формате YYYY-MM-DD. Если не указана, фильтрация по дате не применяется.

page
integer >= 1
Default: 1
Example: page=1

Номер страницы

pageSize
integer [ 1 .. 100 ]
Default: 10
Example: pageSize=10

Размер страницы

Создать новую транзакцию

Создает новую транзакцию с возможностью настройки повторения

Authorizations:
bearerAuth
Request Body schema: application/json
required
amount
required
number <float> >= 0

Сумма транзакции (всегда положительная)

title
required
string non-empty

Название транзакции

category
required
string non-empty

Категория транзакции. 'Доходы' для доходов, остальные для расходов

date
required
string <date>

Дата транзакции в формате YYYY-MM-DD

repeatTime
string

Повторение транзакции (дни недели и числа месяца). Опциональный параметр.
